09:16 AM | Fin: O "View By" e "Segment By" funcionam como dimensões adicionais nas queries, criando agrupamentos nos dados já filtrados por data. (https://www.intercom.com/help/en/articles/8870221-custom-report-chart-types-and-options) 
 View By (Agrupar por): 
 Tecnicamente aplica GROUP BY na query SQL. Por exemplo, se você escolhe "View by Week", a query adiciona GROUP BY WEEK(created_at), criando buckets temporais. (https://www.intercom.com/help/en/articles/8921906-reporting-faqs) 
 Segment By: 
 Adiciona uma segunda dimensão de agrupamento. Com "View by Week" + "Segment by Team", a query fica algo como: 
 GROUP BY WEEK(created_at), team_assignee_id 
 Comportamento técnico importante: 
 Cada conversa conta apenas uma vez por período. Se uma conversa tem múltiplas respostas ao longo de vários dias, ela aparece só uma vez quando agrupada por semana, mas aparece em cada dia quando agrupada por dia. 
 Processamento: 
 1. Query base com filtros de data 
2. Aplica GROUP BY nas dimensões escolhidas 
3. Calcula agregações (COUNT, AVG, SUM) para cada grupo 
4. Retorna dados estruturados para visualização em colunas/segmentos 
 Isso permite visualizar dados em duas dimensões - tempo + atributo específico - mantendo a eficiência da consulta. (https://www.intercom.com/help/en/articles/4549035-create-a-custom-report)
